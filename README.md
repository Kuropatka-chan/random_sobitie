# random_sobitie

## Маршрутная карта проекта

### Основная страница генератора событий
- `generator_lurra_events.html` — UI-оболочка генератора (форма выбора, статус загрузки имен, карточка результата, каталог сущностей).
- `src/events/main.js` — основной контроллер генерации:
  - инициализация селекторов;
  - генерация случайных событий;
  - фильтрация объектов по смысловым тегам действия;
  - подстановка имен в шаблон `[имя]`;
  - копирование результата;
  - обработка решений игрока («Вмешаться» / «Проигнорировать») через d20-бросок с модификаторами;
  - выбор формы последствий с учетом того, может ли источник угрозы выполнить «обратное» действие напрямую.
- `src/events/eventData.js` — структурированная база генератора:
  - наборы (`basic`, `political`, `ecological`, `religious`);
  - лица с родом/числом (`m/f/n/p`);
  - действия с глагольными формами для каждого рода/числа;
  - формы последствий после вмешательства (`reactionForms`) и отдельные формы для неагентных/неодушевленных источников (`nonAgentReactionForms`);
  - объекты и их семантические теги;
  - допустимые семантические теги для каждого действия;
  - флаг `canReverseDirectly` у лиц, которые не могут логично «совершить обратное действие» (например, зараженный колодец).
- `src/events/grammar.js` — грамматическая логика:
  - выбор случайного имени из пула;
  - склонение мужских имен в родительный падеж (для шаблонов типа `графа [имя]`);
  - замена `[имя]` с учетом нужного падежа;
  - сбор имен из `medieval_name_generator_ru_male_unique_v2.json`.

### Файлы данных
- `medieval_name_generator_ru_male_unique_v2.json` — источник имен и фамилий (используются мужские имена из всех культур).
- `data/events_entities_actions_ru.txt` — отдельный текстовый список всех лиц, действий и объектов для ручного редактирования в любом текстовом редакторе.

### Генератор имен (вспомогательный интерфейс)
- `medieval_name_generator_ui.html` — отдельный UI для тестирования и сборки имен/фамилий из JSON.

## Алгоритм генерации события
1. Выбирается набор (`dataset`).
2. Выбираются лицо и действие.
3. Для действия вычисляется список **только совместимых объектов** по тегам (исключаются нелогичные комбинации).
4. Для лица берется правильная форма глагола по роду/числу.
5. Если в лице есть `[имя]`, имя подставляется из JSON-пула; при необходимости — склоняется (родительный падеж).
6. Собирается финальная фраза в формате: `Лицо + действие(согласованное) + объект`.

## Алгоритм исхода после решения игрока
1. Игрок выбирает «Вмешаться» (+2 к броску) или «Проигнорировать» (-2 к броску).
2. Выполняется бросок d20 и применяется модификатор.
3. При результате `< 11` вмешательство/бездействие не дает положительного исхода.
4. При результате `>= 11` используется форма последствий действия:
   - обычная обратная форма (`reactionForms`) — для агентных лиц;
   - форма прекращения действия (`nonAgentReactionForms`) — для неагентных/неодушевленных источников.

Это убирает нелогичные фразы вроде «зараженный колодец очистил ...»: вместо этого выводится «... перестал заражать/загрязнять ...».

## Что можно редактировать вручную
1. Полный список текстовых сущностей: `data/events_entities_actions_ru.txt`.
2. Системные данные и теги: `src/events/eventData.js`.
3. Правила склонения: `src/events/grammar.js`.
